WMN Scheduling Analysis ENG

The project consists of the analysis and design of some scheduling algorithms for Multi-Hop Wireless Mesh Networks in Time Division Multiple Access mode.
The work can be divided into three sections: ETT, WCETT, WCSLOT.
ETT (Expected Transmission Time):
- analysis of the metric;
- implementation of the metric in the network simulator;
- evaluation of the metric.
WCETT (Weighted Cumulative ETT):
- analysis of the metric.
WCSLOT (Weighted Cumulative SLOT):
- design of a new metric (research phase);
- implementation of the metric in the network simulator;
- evaluation of the metric.
Used programming languages: C++, Python, Perl, XML.
The project was realized in partnership with:
Alessandro Pischedda (Website - alessandro.pischedda[at]gmail.com),
Luca Laudadio (laudadio.luca[at]gmail.com),
Silvio Bianchi (silv.bianchi[at]gmail.com).
The source code can not be published due to copyright restrictions on the network simulator.
The project presentation (in English) is available at this link and it can be found in the downloads section.

Secure Crypto ITA

The project consists of a system that implements a secure cryptographic protocol into a distributed Client-Server type application.
The Server owns a couple of keys: a public key and a related private key.
The Server's public key is known to the Client and the Client shares a secret password with the Server.
The Client generates the symmetric session key.
The encryption functions are realized using the OpenSSL tool and the related extended libraries for the C++.
Asymmetric cipher:
- RSA encryption with DES encoding;
- key length: 1024 bit.
Symmetric cipher:
- block encryption in ECB mode;
- block size: 64 bit.
Used programming languages: C++.
The comments and the documentation are in Italian language.
All the project files, with formal specifics, are available in a compressed folder in the downloads section.

Distributed Memory ITA

The project consists of a system that implements a distributed memory abstraction.
The memory is divided into blocks of fixed sizes and each of them is identified by an ID number.
Each Server process provides a default number of blocks.
The number of Servers and the blocks serverd by the pertinet Server are contained in a config file that specifies for each Server the list of the managed ID.
The communication between Clients and Servers and the management of the memory blocks is done using interface functions provided by a purpose-built library.
Main library functions:
- library initialization;
- mapping of global memory addresses on local memory addresses;
- unmapping of global memory addresses on local memory addresses;
- updating of the local block copy with the global content on the Server;
- synchronization of the local block copy with the global content of the reference Server;
- validity check of the local block copy.
Used programming languages: C++.
The project was realized in partnership with Silvio Bianchi (silv.bianchi[at]gmail.com).
The comments and the documentation are in Italian language.
All the project files, with formal specifics, are available in a compressed folder in the downloads section.

Active Monitors ITA

The project consists of an extended synchronization mechanism between processes similar to rendez-vous mechanism.
The application implements a class of entries with specific methods that allow a Server to accept different requests from multiple Clients.
Low-level tools (synchronized, wait, notify and notifyAll methods) were used to manage the synchronization.
Possible operations:
- single request for a resource;
- single release of a resource;
- double request for a resource;
- double release of a resource.
There are two types of Server: one with and one without priority.
The Server with Priority prefers double requests and double releases to single requests and single releases.
Used programming languages: Java.
The project was realized in partnership with Silvio Bianchi (silv.bianchi[at]gmail.com).
The comments and the documentation are in Italian language.
All the project files, with formal specifics, are available in a compressed folder in the downloads section.

Webserver ITA [Basic Version]

Concurrent multi-thread webserver.
The concurrency is obtained with the implementation of POSIX Threads.
The server is able to serve HTTP version 1.0 requests.
Specifically the server supports only full type requests using GET method like:
GET /index.html HTTP/1.0.
Supported MIMEs:
- /text/html, for HTML files with htm e html extensions;
- /text/css, for CSS files with css extensions;
- /text/js, for JAVASCRIPT files with js extensions;
- /text/plain, for TEXT files with txt extensions;
- /image/jpeg, for JPEG images with jpg e jpeg extensions;
- /image/png, for PNG images with png extensions;
- /image/gif, for GIF images with gif extensions.
Error management:
- 404, in the case that the request file doesn't exist;
- 501, in the case that the server can't satisfy the request.
The application can be used with the same compatibility in UNIX and BSD environments.
Used programming languages: C.
The comments are in Italian language.
